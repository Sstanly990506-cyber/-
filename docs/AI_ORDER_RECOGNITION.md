# AI 工單識別設定

AI 工單識別會讓管理員或作業人員拍照／上傳工單，將識別結果填入既有工單表單。結果不會自動儲存，使用者必須確認後手動儲存。

## 環境變數

- `OPENAI_API_KEY`：必要。只設定於伺服器或 Vercel，不可放入前端程式。
- `OPENAI_ORDER_MODEL`：選填。預設為 `gpt-5.4-mini`。

未設定 `OPENAI_API_KEY` 時，原本手動建立工單功能不受影響，AI 識別會顯示設定提示。

在 Vercel 的 Project Settings → Environment Variables 新增 `OPENAI_API_KEY` 後，必須重新部署正式站才會生效。

## 限制

- 支援 JPEG、PNG、WebP。
- 手機端會自動多次壓縮圖片，確保請求低於 Vercel 的 4.5 MB 上限。
- AI 識別最長允許執行 60 秒。
- AI 可能辨識錯誤，儲存前必須人工確認所有欄位。

## 上光廠判讀規則

- 工單最上方、字體最大的完整公司名稱通常是「客人」；若有明確標示的欄位，則以欄位為準。
- 「客人」與「上游客戶」分開判讀，不再自動設成同一家公司。
- `廠商`、`供應商`、`印刷` 等欄位中的印刷廠通常是上游客戶，也就是把紙交給三青的公司。
- 三青／上光後面的 `軋盒`、`軋工`、`軋合`、`裁切`、`軋型` 等下一站通常是下游客戶。
- `三青` 或 `上光` 列代表本公司，不當作上游或下游客戶。
- 日期只讀取交貨日期；若只有發單日期，交貨日期留白。
- 工單編號只讀取 `工單編號`、`工單號碼`、`工單NO` 等欄位，不使用客戶代號。
- 地址代表完成上光後要送往下一站的送貨地址，必須屬於下游客戶。
- 客戶系統有下游地址時一律優先使用；不得使用表頭、發單公司、廣告設計公司、上游印刷公司或上光廠地址。
- 數量優先讀取 `三青` 欄位。「數量說明」視為備註，完整保留中文、符號與原始寫法，例如 `1362車+238張`。
- 「計算張數」只填單據明確寫出的真正數量；不會自行加總數量算式，沒有明確總數時填 `0`。
- `天` 對應表單的天／長，`地` 對應表單的地／寬，不可因數字大小而交換。
- 尺寸單位必須依原稿填寫：`台吋`、`mm`、`cm`、`英吋／吋`；沒有看到單位時留白。
