# 容量與效能

## 大量資料模式

正式環境建議使用 PostgreSQL。系統會將每筆工單、客戶、庫存、稽核、應收、應付與事件分開儲存，不再每次讀寫整包 JSON。

前端每頁預設只載入 100 筆，API 單次上限為 500 筆；背景同步每 10 秒只取得有變更的紀錄。報表由後端計算，避免瀏覽器載入全部資料後才統計。

## 建議容量

- 本機 JSON：適合開發、展示與少量資料，不建議作為多人或大量資料的正式環境。
- PostgreSQL：適合正式環境與數萬筆以上資料。實際上限取決於資料庫規格、同時使用人數與索引狀況。
- 單頁 100 筆：可控制瀏覽器記憶體與表格渲染時間，避免資料增加後畫面明顯卡頓。

## 驗證方式

```bash
python -m unittest tests.test_records -v
```

此測試會驗證 520 筆資料的分頁、API 單次 500 筆上限、搜尋、單筆新增、刪除與增量同步。正式上線前應另以接近真實資料量的測試資料，在部署環境進行壓力測試並監控 API 延遲、資料庫 CPU、記憶體與儲存空間。
